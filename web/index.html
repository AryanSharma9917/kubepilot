KubePilot is an agentic Kubernetes operations copilot. It combines FastAPI, RAG over runbooks, Kubernetes tool-calling, incident reports, metrics, audit events, and tracing into a local-first demo platform.

About the project

What KubePilot Actually Is

KubePilot is an agentic AI Kubernetes operations workspace built to make production troubleshooting faster, clearer, and more explainable. Instead of jumping between dashboards, terminals, runbooks, and incident notes, an engineer can ask one operational question and get a guided answer backed by cluster evidence.

The project was built to demonstrate how AI can support real DevOps and SRE workflows without hiding the underlying facts. KubePilot keeps the Kubernetes tool boundary explicit, retrieves runbook context, shows diagnosis details, and produces incident-ready reports that a human can review before taking action.

Why It Was Built

  • To reduce the time spent manually stitching together pods, events, logs, and runbooks.
  • To show how an AI agent can explain Kubernetes failures with evidence.
  • To create a portfolio-grade DevOps project that combines backend, AI, Kubernetes, and UI work.
  • To keep troubleshooting local-first and demo-friendly while still supporting a real cluster boundary.

Built By

Aryan Sharma DevOps / SRE focused engineer building agentic AI tools for infrastructure operations.

KubePilot is designed as a practical showcase of Kubernetes automation, observability, retrieval, incident reporting, and AI-assisted operations.

Languages And Stack

What It Does
  • Answers operational questions with runbook-backed context.
  • Inspects cluster health through Kubernetes tool boundaries.
  • Diagnoses deployments with pods, events, logs, and recommendations.
  • Generates incident reports as JSON and markdown.
How The Agent Works
  1. Classifies the user intent.
  2. Retrieves matching runbook context.
  3. Calls Kubernetes health or diagnosis tools when needed.
  4. Synthesizes an answer with evidence and next actions.
  5. Records audit events, metrics, and trace spans.
